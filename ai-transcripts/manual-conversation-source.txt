# 5. Extracting this conversation from `conversations.json`

You can use a small Node.js script.

Create:

```text
scripts/extract-ai-conversation.cjs
```

Add:

```javascript
const fs = require("fs");
const path = require("path");

const inputPath = process.argv[2];
const keyword = process.argv[3];
const outputPath =
  process.argv[4] || "ai-transcripts/chatgpt-autocomplete-assignment.json";

if (!inputPath || !keyword) {
  console.error(
    'Usage: node scripts/extract-ai-conversation.cjs "<conversations.json>" "<keyword>" "[output.json]"'
  );
  process.exit(1);
}

if (!fs.existsSync(inputPath)) {
  console.error(`Input file not found: ${inputPath}`);
  process.exit(1);
}

const exportedData = JSON.parse(fs.readFileSync(inputPath, "utf8"));
const conversations = Array.isArray(exportedData)
  ? exportedData
  : [exportedData];

const normalizedKeyword = keyword.toLowerCase();

const matches = conversations.filter((conversation) =>
  JSON.stringify(conversation)
    .toLowerCase()
    .includes(normalizedKeyword)
);

if (matches.length === 0) {
  console.error(`No conversation found containing: ${keyword}`);
  process.exit(1);
}

fs.mkdirSync(path.dirname(outputPath), { recursive: true });

fs.writeFileSync(
  outputPath,
  JSON.stringify(matches, null, 2),
  "utf8"
);

console.log(`Found ${matches.length} matching conversation(s).`);
console.log(`Saved to: ${outputPath}`);
```

Run it using a phrase unique to the assignment:

```powershell

  "Autocomplete Form Requirements"
```

Then inspect:

# 8. Understand the supplied API discrepancies

Given:

```json
{
  "account_id": "98765",
  "account_email": "test123@gmail.com",
  "start_date": "2024-03-15T10:30:00Z",
  "end_date": "2024-03-15T10:32:00Z",
  "locale": "en",
  "text": "agile methodology",
  "suggestion_list": "agile methodology, agile methodology process, agile methodology process testing",
  "completed": "true"
}
```

## Confirmed problem 1: `completed`

Given:

```json
"completed": "true"
```

This is a string.

Required:

```json
"completed": true
```

That is a Boolean.

## Confirmed problem 2: local timestamp requirement

`Z` means UTC:

```text
2024-03-15T10:30:00Z
```

The requirement says timestamps should be in the user’s local time. For India, an explicit local representation would normally contain:

```text
+05:30
```

For the same instant:

```text
2024-03-15T16:00:00+05:30
```

This should also be recorded as a requirement clarification because many production APIs deliberately persist UTC even when the UI displays local time. But FR-05 explicitly says local time.

## Locale nuance

Given:

```json
"locale": "en"
```

`en` is a valid BCP 47 language tag.

However, based on the supplied environment—English in India—the expected environment-specific locale would be:

```json
"locale": "en-IN"
```

Therefore, do not claim that `en` is syntactically invalid. Say that it is valid but insufficiently specific for the stated environment.

## `suggestion_list` is not necessarily defective

All three values begin with:

```text
agile methodology
```

Therefore, under prefix matching, all three are valid matching suggestions.

This is an important example where you corrected overly simplistic AI output.

# What the whole system is doing

Before analysing individual requirements, understand the complete flow.

```text
User opens form
       ↓
Browser records start_date
       ↓
User types in the input
       ↓
Filtering logic calculates matching suggestions
       ↓
Matching suggestions are displayed
       ↓
User types an exact value or selects a suggestion
       ↓
User clicks Next or presses Enter
       ↓
Frontend validates the value
       ↓
Invalid → show error
Valid   → create API payload
       ↓
POST request sends the response to backend
       ↓
Backend validates and persists it
       ↓
HTTP 200 returned
       ↓
Frontend shows success message
       ↓
GET API can retrieve persisted response
       ↓
API automation validates FR-05
```

The UI and API are therefore not independent.

The UI determines:

- What the user typed
- What suggestions matched
- When the user arrived
- When the user submitted
- What payload was sent

The backend determines:

- Whether the payload is accepted
- Whether it is stored correctly
- What data types are returned
- Whether the persisted information matches the user’s action

# FR-01: Text Input

## Original requirement

> Users can type any response in the text field OR click/tap a suggestion list item to select it.

FR-01 contains two separate capabilities.

### Capability A: Free typing

The input must accept text typed by the user.

Examples:

```text
agile
```

```text
agile methodology
```

```text
not a valid suggestion
```

“Users can type any response” means the input should accept the text. It does **not** necessarily mean that every typed value can be successfully submitted.

That distinction is essential:

```text
Input acceptance ≠ submission validity
```

A user may be allowed to type:

```text
water management
```

but the form may reject it when Next is selected.

This connects FR-01 with FR-04:

```text
FR-01 allows entry
FR-04 decides whether the entry is valid for submission
```

---

### Capability B: Suggestion selection

The user can click a visible suggestion.

For example:

```text
agile methodology process
```

After clicking, the input value must become exactly:

```text
agile methodology process
```

It should not become:

```text
agile methodology
```

or:

```text
Agile Methodology Process
```

unless the requirement defines a transformation.

---

## FR-01 testable conditions

We need to verify:

1. The input is visible.
2. The input is enabled.
3. The input is editable.
4. Arbitrary text can be entered.
5. A suggestion can be clicked.
6. The complete selected suggestion populates the input.
7. Clicking one suggestion does not populate another suggestion.
8. The input remains usable after selection.

---

## FR-01 ambiguities

The requirement does not define:

- Maximum input length
- Minimum input length
- Special-character handling
- Case sensitivity
- Leading and trailing spaces
- Pasting text
- Unicode input
- Whether mobile touch must be tested
- Whether free text can be submitted successfully

We should not silently invent these rules.

For this assignment, we can document:

> The field accepts arbitrary typed text, but based on FR-04’s invalid-input message, successful submission is assumed to require an exact configured suggestion.

That is an **assumption**, not a confirmed product rule.

---

## FR-01 code mapping

### HTML input

```html
<input type="text" id="input-field" placeholder="Type here...">
```

### Page Object locator

```typescript
this.input = page.locator('#input-field');
```

### Page Object typing method

```typescript

### Page Object suggestion-selection method

```typescript
async selectSuggestion(text: string): Promise<void> {
  await this.suggestionItems
    .filter({ hasText: text })
    .getByText(text, { exact: true })
    .click();
}
```

`exact: true` prevents accidentally selecting a longer suggestion containing the same words.

For example, it prevents this request:

```text

```

from accidentally clicking:

```text
agile methodology process
```

const fs = require("fs");

const messages = [

  },
  {

  }
];

const transcript = {
  transcript_metadata: {
    title: "SatSure Autocomplete Form Practical Assignment",
    ai_tool: "ChatGPT",
    capture_method: "Manual capture from the user-visible conversation",
    official_platform_export: false,
    scope: "Only messages directly related to the SatSure autocomplete assignment",
    completeness_statement:
      "The messages are recorded in chronological order without intentionally removing failed attempts, corrections, debugging exchanges, or implementation discussions."
  },
  messages
};

const outputPath = path.join(
  __dirname,
  "..",
  "ai-transcripts",
  "chatgpt-assignment-visible-transcript.json"
);

fs.mkdirSync(path.dirname(outputPath), { recursive: true });

fs.writeFileSync(
  outputPath,
  JSON.stringify(transcript, null, 2),
  "utf8"
);

console.log(`Transcript created: ${outputPath}`);
console.log(`Messages included: ${messages.length}`);
```

JavaScript template strings use backticks:

```javascript
content: `multiple
lines can be pasted
without \n escaping`
```

This is safer than editing complex JSON directly.

Do not include a backtick character inside the pasted message without escaping it. Most of our conversation does not contain backticks visibly, but code blocks may. When we reach those responses, we can handle them carefully.

## `type`

```json
"type": "module"
```

This tells Node.js that normal `.js` files use the modern ES module system.

ES module syntax looks like:

```typescript
import { test, expect } from '@playwright/test';
```

Older CommonJS syntax looks like:

```javascript
const fs = require('fs');
```

The test project uses modern imports.

However, the mock server is named:

```text
mock-server.cjs
```

The `.cjs` extension explicitly tells Node:

> Treat this particular file as CommonJS even though the overall project uses ES modules.

That is why `mock-server.cjs` can use:

```javascript
const http = require('http');
```

while TypeScript files can use:

```typescript
import { test } from '@playwright/test';
```

This is an important connection:

```text
"type": "module"
        ↓
.js files default to ESM
        ↓
.cjs used when CommonJS is required
```

## `ajv`

```json

```

AJV stands for **Another JSON Schema Validator**.

It validates an API response against a JSON schema.

For example, the schema says:

```typescript

}
```

AJV checks whether the response contains:

```json
"completed": true
```

rather than:

```json

```

AJV mainly validates **structure and declared formats**.

It does not automatically understand all business meaning.

For example, schema validation alone may not know that:

```text
end_date must be after start_date
```

That requires a separate semantic assertion.

# Modification 3: Prove invalid input sends no POST request

Find the current final test:

```typescript
test('UI-009: Invalid free text shows an error and no success state', async ({ page }) => {
  const form = new AutocompleteFormPage(page);
  await form.open();
  await form.typeValue('not a configured suggestion');

  await form.submitByClick();

  await expect(form.errorMessage).toContainText('Invalid input');
  await expect(form.errorMessage).toBeVisible();
  await expect(form.successMessage).toBeHidden();
});
```

Replace the complete test with:

```typescript
test('UI-010: Invalid free text shows an error and sends no persistence request', async ({ page }) => {
  const form = new AutocompleteFormPage(page);
  let submissionRequestCount = 0;

  page.on('request', request => {
    if (
      request.url().includes('/api/responses') &&
      request.method() === 'POST'
    ) {
      submissionRequestCount += 1;
    }
  });

  await form.open();
  await form.typeValue('not a configured suggestion');
  await form.submitByClick();

  await expect(form.errorMessage).toContainText('Invalid input');
  await expect(form.errorMessage).toBeVisible();
  await expect(form.successMessage).toBeHidden();
  expect(submissionRequestCount).toBe(0);
});
```

## What is happening here?

We start with:

```typescript
let submissionRequestCount = 0;
```

This variable counts persistence requests.

Playwright listens to every browser request:

```typescript
page.on('request', request => {
```

We only count requests that:

- Go to `/api/responses`
- Use the HTTP `POST` method

```typescript
if (
  request.url().includes('/api/responses') &&
  request.method() === 'POST'
) {
  submissionRequestCount += 1;
}
```

After invalid input is submitted, we assert:

```typescript
expect(submissionRequestCount).toBe(0);
```

That proves the frontend rejected invalid data before it reached the backend.

The expected system flow is:

```text
Invalid input
    ↓
Frontend validation fails
    ↓
Error displayed
    ↓
submit() returns early
    ↓
No fetch()
    ↓
No POST request
```

You can see that in `mock-app/app.js`:

```javascript
if (!isExactSuggestion) {
  errorMessage.style.display = 'inline';
  return;
}
```

Because of `return`, this later code never runs:

```javascript
const response = await fetch('/api/responses', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
});
```

That connects:

```text
Requirement → frontend implementation → Playwright assertion
```

Paste this code:

```javascript
const fs = require("fs");
const path = require("path");

const projectRoot = path.resolve(__dirname, "..");

const sourcePath = path.join(
  projectRoot,
  "ai-transcripts",
  "manual-conversation-source.txt"
);

const transcriptPath = path.join(
  projectRoot,
  "ai-transcripts",
  "chatgpt-assignment-visible-transcript.json"
);

const promptJsonPath = path.join(
  projectRoot,
  "prompts",
  "all-user-prompts.json"
);

const promptMarkdownPath = path.join(
  projectRoot,
  "prompts",
  "all-user-prompts.md"
);

function fail(message) {
  console.error(`ERROR: ${message}`);
  process.exit(1);
}

if (!fs.existsSync(sourcePath)) {
  fail(`Source file not found: ${sourcePath}`);
}

const source = fs.readFileSync(sourcePath, "utf8").replace(/\r\n/g, "\n");

const messagePattern =
  /<<<MESSAGE role="(user|assistant)">>>\n([\s\S]*?)\n<<<END_MESSAGE>>>/g;

const messages = [];
let match;

while ((match = messagePattern.exec(source)) !== null) {
  const role = match[1];
  const content = match[2];

  if (!content.trim()) {
    fail(`Empty message detected after sequence ${messages.length}.`);
  }

  messages.push({
    sequence: messages.length + 1,
    role,
    content
  });
}

if (messages.length === 0) {
  fail("No valid transcript messages were found.");
}

const unmatchedSource = source
  .replace(messagePattern, "")
  .trim();

if (unmatchedSource.length > 0) {
  console.warn(
    "WARNING: Text was found outside the message delimiters. " +
      "Review manual-conversation-source.txt."
  );
}

const userMessages = messages.filter(message => message.role === "user");

const transcript = {
  transcript_metadata: {
    title: "SatSure Autocomplete Form Practical Assignment",
    ai_tool: "ChatGPT",
    capture_method:
      "Manually captured from the user-visible ChatGPT conversation",
    official_platform_export: false,
    scope:
      "Assignment-related messages only, from the original assignment prompt through final submission preparation",
    generated_from:
      "ai-transcripts/manual-conversation-source.txt",
    completeness_statement:
      "The transcript preserves the assignment-related visible conversation in chronological order, including failed setup attempts, corrections, debugging, requirement analysis, code review and submission preparation.",
    message_count: messages.length,
    user_message_count: userMessages.length,
    assistant_message_count: messages.length - userMessages.length
  },
  messages
};

const prompts = {
  prompt_metadata: {
    ai_tool: "ChatGPT",
    source_transcript:
      "ai-transcripts/chatgpt-assignment-visible-transcript.json",
    extraction_rule:
      "Every transcript message whose role is user is included without rewriting.",
    prompt_count: userMessages.length
  },
  prompts: userMessages.map((message, index) => ({
    prompt_number: index + 1,
    transcript_sequence: message.sequence,
    content: message.content
  }))
};

const promptMarkdown = [
  "# Complete User Prompt Record",
  "",
  "- AI tool: ChatGPT",
  "- Capture method: Extracted from the manually captured transcript",
  "- Editing applied: Headings only; prompt wording unchanged",
  `- Prompt count: ${userMessages.length}`,
  "",
  ...userMessages.flatMap((message, index) => [
    `## Prompt ${index + 1}`,
    "",
    `Transcript sequence: ${message.sequence}`,
    "",
    "```text",
    message.content,
    "```",
    ""
  ])
].join("\n");

fs.mkdirSync(path.dirname(transcriptPath), { recursive: true });
fs.mkdirSync(path.dirname(promptJsonPath), { recursive: true });

fs.writeFileSync(
  transcriptPath,
  JSON.stringify(transcript, null, 2),
  "utf8"
);

fs.writeFileSync(
  promptJsonPath,
  JSON.stringify(prompts, null, 2),
  "utf8"
);

fs.writeFileSync(
  promptMarkdownPath,
  promptMarkdown,
  "utf8"
);

console.log("AI evidence generated successfully.");
console.log(`Messages: ${messages.length}`);
console.log(`User prompts: ${userMessages.length}`);
console.log(
  `Assistant messages: ${messages.length - userMessages.length}`
);
console.log(`Transcript: ${transcriptPath}`);
console.log(`Prompt JSON: ${promptJsonPath}`);
console.log(`Prompt Markdown: ${promptMarkdownPath}`);
```

## What this script does

It reads:

```text
manual-conversation-source.txt
```

It converts every delimited message into:

```json
{
  "sequence": 1,
  "role": "user",
  "content": "Exact message"
}
```

It then automatically extracts every user message into:

```text
prompts/all-user-prompts.json
prompts/all-user-prompts.md
```

Therefore:

```text
Transcript user messages = Submitted prompts
```

You do not have to manually synchronize the two.
